What is Octave?

  • Extendable scripting language
  • Intended for numerical computations
  • For solving linear and nonlinear problems numerically and for performing other numerical experiments
  • Mostly compatible with Matlab ®
  • Interpreter provides a convenient command line interface (using the familiar libreadline)

What is Octave? (cont'd)

  • Main engine contains tools for
    • Solving linear algebra problems
    • Finding roots of non-linear equations
    • Integrating ordinary functions, ODEs etc.
    • Manipulating polynomials
    • And many, many more...

What is Octave? (cont'd)

  • Toolboxes for solving problems in
    • Signal processing, Data analysis, Calculus, Finances, Symbolic algebra, Time-series analysis, Communications Theory, Control Theory, Image Processing, Geometry, Mapping, Optimisation, Statistics, Fixed-point calculations
    • and many, many more...

What is Octave? (cont'd)

  • GNU Octave is free software.
  • Also “Open Source” but authors strongly prefer “free software”
    • Free as in beer and as in freedom – liberté!
    • Free to modify, distribute, add, contribute
    • Freedom is guaranteed and protected by the GNU General Public License

History

  • 1988: Companion software for chemistry textbook
    • Fortran wasted students' time debugging (think C++ ...)
  • 1992: Development started • 1993: First alpha release
  • 1994, February 17: Version 1.0
  • From courseware to MATLAB competitor in 10 years, now used by thousands for scientific research
  • Named after one of the author's professors who did “back of the envelope” calculations with ease.

The case for MATLAB®

  • High-quality, complete, supported
  • Wide variety of toolboxes, platforms
  • Expensive. Individual academic license (July 2004) roughly
    • R5200 for MATLAB engine
    • R2200 per toolbox

The case for Octave

  • Much of the functionality of MATLAB
  • Extra toolboxes supplied by external projects like Octave Forge (25+ toolboxes)
  • Free (as in speech) and free (as in beer)
  • Community support on a very active mailing list
  • Well supported on Linux and MacOS platforms

Speed comparisons

  • Differs much, depending on functions tested
  • Octave doesn't yet do JIT compiling
    • Some progress recently
  • Slow execution often result of bad coding (vectorise, vectorise, vectorise!)
  • Not the ideal, but slow performance can often be remedied by recoding M-functions in C++

Possible Octave issues

  • Windows platform supported via Cygwin
  • Dependent on volunteers and quality of their contributions
  • Dependent on donations to fund development
  • Some want a graphical interface
  • Mimics MATLAB, luckily not always second

Extending Octave in C++

  • #include <oct.h>
  • DEFUN_DLD(mtest, args, nargout, "my test func")
  • {
  • Matrix M(3, 3, 0);
  • for (int i=0; i < 3; i++) { M(i,i) = i; }
  • octave_value_list retval; retval.append(M); return retval;
  • }
